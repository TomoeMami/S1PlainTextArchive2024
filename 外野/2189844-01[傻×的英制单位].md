在英制系统下，板材厚度通常用“Gauge”(缩写为“Ga.”) 来表示。这个单位源于铁丝的直径的表示，因此，在铁制品方面，Gauge 可以直接用来表示其厚度。
1. 铁质板材的 Gauge 表示方法
10 Gauge 的铁板厚度为 0.1345 英寸，后续的 Gauge 数值按照每增加 1，厚度减小 0.0005 英寸的规律递减。例如，9 Gauge 的铁板厚度为 0.1495 英寸，8 Gauge 的厚度为 0.1644 英寸，以此类推。
2. 不锈钢板材的 Gauge 表示方法
不锈钢板材的 Gauge 表示方法与铁质板材相似，不同之处在于不锈钢板的厚度比铁板的厚度要小。例如，10 Gauge 的不锈钢板厚度为 0.1406 英寸，而 9 Gauge 的不锈钢板厚度为 0.1562 英寸。

这个说法实际上应该反过来，摄氏度本质是开尔文温标的非线性变换。开尔文在设计上确实基于实用性的考虑，参照了摄氏度的分度值 1 ΔK = 1Δ℃。不过这只是一种选择，如果选择参考华氏度，也有对应的热力学温标Rankine scale， 1Δ°R = 1Δ℉，只是最后还是开尔文上位当了热力学温度的国际标准。

但是零度的定义换了，意味着人对“温度”的进一步理解，0K有更普适的物理意义，是分子热运动的最低状态，可以更加严格地进行物理计算。相比之下，受限于当时的科学发展水平，0℃和0℉只是特定物质特定状态下的一个特殊值。

考虑一个场景

1℃ + 2℃ = 3℃

然后做一下代换

1℃ = 274.15K

2℃ = 275.15K

3℃ = 276.15K

那么就有

274.15K + 275.15K = 276.15K ？

在测量里，摄氏华氏温标没有一个绝对零点，都是间隔尺度（interval scale), 可以计算和比较差值，但不可以直接将两个测量值相加。就是华氏摄氏上 没有定义加法，开氏兰氏温标才是有零点可以定义加法的比率尺度（Ratio Scale）。上面的式子其实应该写成1℃ + 2Δ℃ = 3℃

受限于当时的科学水平，老温标有它的局限性，但是确实更贴近日常生活，方便是真方便，本质还是热力学温标更本质。

####  只要主义真  
##### 196#       发表于 2024-7-5 15:53

<blockquote><a href="httphttps://bbs.saraba1st.com/2b/forum.php?mod=redirect&amp;goto=findpost&amp;pid=65482996&amp;ptid=2189844" target="_blank">shamisen 发表于 2024-7-4 21:57</a>

有什么场景可以1c+2c呀？</blockquote>
摄氏度是间隔尺度，没法加。

当我们在自然语言中描述“把一瓶1摄氏度的水，加热上升2摄氏度，变成一瓶3摄氏度的水“的时候，实际表达的是1摄氏度 + 2个摄氏度差值 = 3摄氏度。

理论上，减法是定义在加法上的逆运算，没法加肯定也没法减。但是在计算温度变化放热的时候经常会用到两个摄氏度相减乘以比热。比如说计算一公斤水从2摄氏度降低到1摄氏度会放出多少热量。

这里的减法实际是跑在开尔文上的。由于前面说到开氏温标沿用了摄氏温标的分度值，摄氏温标刚好是来自开尔文温标的一个只有简单平移的非线性变换。

1℃ = (273.15 + 1) K 

2℃ = (273.15 + 2)K

两边一减吧273.15的offset减掉了，结果的温差刚好等于摄氏度的数字差，给人一种摄氏度可以直接减的错觉

2℃ - 1℃ 没有定义，不能算

(273.15 + 2)K -  (273.15 + 1) K 可以算 = 1K

这里我强行规定一个摄氏度减法。2℃ - 1℃  = 1℃，再把水的比热容从4200J/( kg · K )掰成4200J/( kg · ℃ ) （摄氏度没有加减，自然也也没有乘除）。就可以说

（2℃ - 1℃）* 4200J/( kg · ℃ ) * 1kg 

= 1℃ * 4200J/( kg · ℃ ) * 1kg

= 4200J

过程错误，答案正确，这里就很有一种程序跑在bug上的美感。哪天有人想改bug，但是只改了一半，就会变成

（2℃ - 1℃）* 4200J/( kg · K ) * 1kg

= 1℃ * 4200J/( kg · K ) * 1kg

= 274.15K * 4200J/( kg · K ) * 1kg

= 1151430J

祖传代码，能跑就不要乱动。
